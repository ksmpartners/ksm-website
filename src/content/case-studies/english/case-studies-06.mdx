A client in the pharmaceutical industry engaged KSM to prepare a machine learning/artificial intelligence data engineering proof of concept for production use. The product uses natural language processing to categorize textual data by product, topic, and sentiments, and allows the client to detect emerging signals and preemptively answer questions from healthcare providers and patients.

## Challenge [.text-h3]

---

The client needed a production-ready machine learning and AI data engineering solution that could process textual data using natural language processing. The solution required categorization by product, topic, and sentiment, with the ability to detect emerging signals and enable proactive responses to healthcare providers and patients. The system needed to be deployed reliably with proper quality control and infrastructure management.

## Solution [.text-h3]

---

The project was founded on Azure Data Factory (ADF) and Databricks, with data stored in an Azure Data Lake and SQL server. KSM integrated ADF and Databricks with Azure DevOps to facilitate code review quality checks and pipeline releases. We implemented infrastructure-as-code with Liquibase to ensure structural database changes were delivered by a consistent, reproducible, and reversible process. We instrumented the code with logging and monitoring, and deployed to Azure Kubernetes Service.

## Results [.text-h3]

---

- **Production-Ready ML/AI**: Successfully prepared proof of concept for production use
- **Natural Language Processing**: Categorization of textual data by product, topic, and sentiment
- **Signal Detection**: Ability to detect emerging signals and preemptively answer questions
- **Reliable Deployment**: Enhanced quality control over development operations
- **Infrastructure-as-Code**: Consistent, reproducible, and reversible database changes with Liquibase
- **Continuous Integration**: Azure DevOps integration for code review and pipeline releases
- **Monitoring**: Comprehensive logging and monitoring instrumentation
